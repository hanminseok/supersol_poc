* Project 개요
- 이 프로젝트는 은행App 내에서 고객이 채팅을 통해 업무를 처리하게 도와주는 WebUI 채팅 서비스.
- 채팅은 Multi Agent 기반으로 동작.
- 프로젝트 root directory는 SuperSOL.
- python venv이름은 solenv. 경로는 SuperSOL/solenv. 
- 서비스를 시작할때 항상 solenv를 활성화하고 시작해줘.
- SuperSOL/src directory는 서비스 Code들을 모아둘꺼야.
- 모든 환경변수들은 Config.py에 작성할꺼야.
- 'SuperSOL/src/.env' file 에는 gemini, openai, deepinfra 의 API Key가 있어.
- Agent의 구조는 Supervisor Agent, Domain Agent, Worker Agent 3단계로 구성되어 있어.
- python file과 class은 항상 1:1이야.
- 모든 Agent와 Tool의 Input/Output을 기록할꺼야.

* Tool 구조
- 조회와 실행 두가지 종류 일을 처리하는 Tool들이 있어.
- 고객정보 조회, 금융정보 조회, 이체 조회, 이체 실행, 계좌 조회, 자동이체 조회, 자동이체 실행, 투자상품 조회, 투자상품 실행, 대출 조회, 대출 실행을 할 수 있는 Tool들이 있어.
- Tool들은 Input과 Output이 있어.
- SuperSOL/Data에 Json Sample Data를 만들어서 조회를 담당하는 Tool들을 동작하게 해줘.
- 실행을 하는 Tool들은 "실행되었습니다"라는 Output을 보여줘.

* Supervisor Agent 구조
- Supervisor Agent는 최상위 결정자이고 OpenAI의 GPT4o를 사용 할꺼야.
- Supervisor Agent가 질문의 Context, Intext, Slot을 추출하고 적합한 Domain Agent 들에게 일을 시켜줘.

* Domain Agent 구조
- Domain Agent는 Deepinfra를 통해 Qwen3-30B-A3B LLM 모델을 사용 할꺼야.
- Domain Agent는 고객정보, 금융정보, 투자상품, 계좌, 이체, 대출 6가지의 업무를 잘 처리하는 Agent.
- Domain Agent를 나중에 추가하여 확장 할 수 있는 구조이어야 해.

* Worker Agent 구조
- Worker Agent는 Deepinfra를 통해 Qwen3-14B LLM 모델을 사용 할꺼야.
- Worker Agent는 고객정보 조회, 금융정보 조회, 이체 조회, 이체 실행, 계좌 조회, 자동이체 조회, 자동이체 실행, 투자상품 조회, 투자상품 실행, 대출 조회, 대출 실행 총 11가지가 있어.
- Worker Agent는 업무처리를 위해서 Tool들을 잘 호출하는 것이 목표야.

* Log 관련
- SuperSOL/src/logger.py 는 logging 을 위한 Class야.
- Logger Class는 Consol, File 두개의 handler를 사용할꺼야.
- log file의 path는 'SuperSOL/logs'.
- format은 "%(asctime)s [%(levelname)-8s][%(name)-15s] %(message)s", datefmt="%Y-%m-%d %H:%M:%S"
- Error가 발생하면 어떤 python file의 몇번째line인지 log에 기록하도록 해줘.
- 모든 Agent와 Tool의 Input/Output을 기록하는 file handler를 만들고 하루에 한번씩 log file을 작성하도록 설정해줘. 파일이름은 'Agent_log_{%Y%m%d}.log'.
- Consol에서 발생하는 모든 Log를 기록하는 file handler를 만들고 하루에 한번씩 log file을 작성하도록 설정해줘. 파일이름은 'Service_log_{%Y%m%d}.log'.
